# EMA Leads Funnel тАФ Test Data

## Identity generation

All test leads are generated by `tests/support/api.ts тЖТ testIdentity()`:

- **Full name:** `E2E Test <label> <nonce>` тАФ obviously synthetic, easy to
  find and purge.
- **Email:** `e2e-<label>-<nonce>@example.com` тАФ the reserved `example.com`
  domain guarantees no real inbox exists.
- **WhatsApp:** `+27` + random 9 digits тАФ canonical `+<10-15 digits>` form.
- **Nonce:** timestamp (base36) + in-process counter тАФ unique per call, so
  duplicate-guard (409 on email/WhatsApp reuse) never trips accidentally
  across runs.

## Default lead payload

`createLead()` posts to the public `POST /api/leads` with:

| Field | Value | Why |
|---|---|---|
| `nationality` | `Nigeria` | Any valid country works |
| `immigrationSituation` | `visa_required` | Required by the API |
| `countryOfResidence` | `Nigeria` | Required for conversion readiness |
| `currentlyInSouthAfrica` | `false` | Outside-SA path |
| `consentAccepted` | `true` | POPIA тАФ mandatory |
| `finalize` | `false` | **Critical:** skips confirmation email/WhatsApp so no real messages are ever sent for test data |

Tests that need other shapes (missing fields, honeypot, bad formats) override
via `createLeadRaw(request, { ...overrides, extra })`.

## Admin credentials

- Seeded demo superadmin: `demo@admin.local` / `ChangeMe!2026`
  (override with `E2E_ADMIN_EMAIL` / `E2E_ADMIN_PASSWORD`).
- RBAC tests create throwaway standard admins named
  `e2e-standard-<timestamp>@example.com` via the superadmin user-management
  API; they are deactivated/cleaned by the test where possible.

## Pipeline walk

`advanceLeadTo()` walks the forward pipeline in order:
`contacted тЖТ engaged тЖТ qualified тЖТ proposal_sent тЖТ ready_for_case тЖТ converted`
and pre-sets `inquiryType` where possible for conversion readiness.

## Cleanup / hygiene

- Tests run against the **development** database only.
- Test rows are identifiable by the `E2E Test` name prefix and
  `@example.com` email domain. To purge:
  `DELETE FROM prelaunch_leads WHERE email LIKE 'e2e-%@example.com';`
  (leads with linked cases must have the case removed first, or archive them
  instead).
- Audit rows (`lead_audit`) are append-only by design and are left in place.
